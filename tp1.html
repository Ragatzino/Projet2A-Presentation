Compléments d'informatique : Quelques notions pour le TP1

Cours

Un grand principe : separation of concern

Version macro en 3 couches.
  • Présentation : tout ce qui se charge de l'affichage
  • Business : c'est le métier de votre application, sa plu value
  • Data : gère le stokage. Plus de détails dans quelques minutes
Chaque couche communique avec celles adjacentes. Une couche à juste à savoir comment elle doit demander une information et ce qu'on lui retourne

User Interface

  • Interface-Homme-Machine qui sera ici la console / Main de votre application
  • Dans la vraie vie : Une application mobile / une page web / l'interface graphique d'un jeu

Business Logic

  • Couche métier de l'application dans laquelle l'on définit les différents objets qui interagiront et qui seront sauvegardés par ailleurs
  • Dans la vraie vie : Une application mobile / une page web / l'interface graphique d'un jeu
  • Elle sera ici également utilisée pour l'appel aux services que nous verrons dans de futurs tps !

Data Access

  • Couche interagissant avec la base de données de l'application
  • C'est ici que sera réalisée la persistance de nos données métier (Avengers ici)
  • Expose classiquement les méthodes CRUD : Create Read Update Delete

Separation of concern : bilan

  • Décomposition d'un programme en modules simples cohérents
  • Les modules exposent des méthodes utilisables par d'autres modules
  • Chaque module doit être une boîte noire pour les autres (ce qui importe c'est les entrées/sorties)
  • Si on garde les mêmes entrées/sorties on peut changer un module sans risque
Module archi != module python Couche c'est quand on a des modules empilés (beaucoup de voc à assimiler d'un coup)
Faire un dessin avec et sans. Expliquer que c'est le boulot d'un objet métier de dire comme il s'affiche et comment on le stock. Pareil c'est pa le boulot d'une vue de faire un calcul. Par contre une vue peut demander.
Bien insiter sur "l'indépendance des couches". Théoriquement si deux personnes travaillent sur 2 couches et se sont mise d'accord sur comment elles communiquent le travail peut se faire en parallèle.

Git

Introduction a Git 1/2

  • Git: Logiciel de version décentralisé
  • Permet un partage efficace et sécurisé du code entre différents utilisateurs
  • Permet une parallélisation des taches de travail en utilisant des branches
  • Versionning pour l'équipe projet
  • Gestions des conflits lors du travail sur les mêmes bouts de code

Introduction a Git 2/2

  • Création facilitées de dépots pour les projet
  • Relecture facilitée du code avec outils d'observation devOps: GitHub/ GitLab
  • Historique du code disponible sur les espaces ou Repository aux travers des commits (messages)
  • Versionning pour l'équipe projet

Récupérer le TP via git bash

  • Cloner un répertoire distant dans un dossier local 
    git clone url-github nom-de-dossier-pour-cloner
  • Ajoute tous les fichiers du répertoire à l'index Git 
    git add .
  • Envoie l'ensemble des fichiers de l'index sur le repository local 
    git commit -m "message de commit"
